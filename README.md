# 1 задание

1. Парсинг входных данных
   └── Принять email-адреса: через CLI-аргументы, файл или интерактивный ввод

2. Валидация формата
   └── Regex-проверка базового формата email (user@domain)

3. DNS MX-проверка
   └── Для каждого уникального домена резолвить MX-записи через dnspython

4. SMTP Handshake (VRFY / RCPT TO)
   ├── Подключиться к MX-серверу с наивысшим приоритетом
   ├── Отправить HELO/EHLO
   ├── Отправить MAIL FROM:<>
   ├── Отправить RCPT TO:<email>  ← код ответа покажет, существует ли ящик
   └── Закрыть соединение (без DATA, письмо НЕ отправляется)

5. Вывод результатов
   └── Таблица: email | домен | MX | SMTP-статус


#   Инструкция по запуску

## 1. Установка зависимости

> pip install dnspython

## 2 Запуск

###  Через аргументы CLI:

> python email_verifier.py user@gmail.com fake@nonexistent-domain-xyz.com admin@example.org

### Из файла (по одному email на строку):

> python email_verifier.py -f emails.txt

### Подробный лог:

> python email_verifier.py -v user@gmail.com


## 3 Пример вывода

```
====================================================================================================
EMAIL                               ДОМЕН                          MX              SMTP
----------------------------------------------------------------------------------------------------
user@gmail.com                      ✔ валиден                      gmail-smtp-i..  ✔ существует (250)
fake@nonexistent-domain-xyz.com     ✗ не найден                    —               — пропущено
admin@example.org                   ⚠ нет MX                       —               — пропущено
====================================================================================================
```

⚠️ Важные замечания

Catch-all серверы	Многие серверы (напр. Microsoft 365) отвечают 250 на любой RCPT TO — это не гарантия существования ящика
Greylisting	Некоторые серверы отвечают 4xx при первом подключении с незнакомого IP — повторите через 5 мин
Блокировки	При массовой проверке ваш IP может быть заблокирован — используйте задержки
Порт 25	Многие хостинги/провайдеры блокируют исходящий порт 25 — скрипт нужно запускать на VPS/сервере
